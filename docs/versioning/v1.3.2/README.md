## 定位

1.3.2 继续 1.3.1 的收尾思路：不引入新能力，只把 1.3 / 1.3.1 期间已经启动、但还没一次性做干净的事情收完。

重点是两条线：

1. 叙事与文案一致性：顶栏"纺织"类比已经改为"搭积木"，但 CSS 类名、部分组件名与品牌资产还没跟上；同时整站 kicker / 章节编号也需要统一一遍。
2. 设置入口与本地偏好：1.3.1 末尾引入了设置下拉、拆分了设置页、增加了 `AppPreferences`，但几项偏好目前只是占位开关，没真正接入界面渲染；首次进入、返回链路等细节也需要校验。

## 目标

- 把"搭积木"叙事从用户可见层继续延伸到代码与资产层面（在不破坏现有构建的前提下）
- 修正首页章节编号重复、kicker 风格不一致等可见但被忽略的小问题
- 让本地偏好里真正被引用的项目能实际起作用，把不会很快落地的项目明确标记为占位
- 把 UserMenu 去除后可能仍残留的 user 相关代码和类型清理干净
- 完成一轮 smoke test，确认 1.3.1 的入口重构和 1.3.2 的清理没有引入回归

## 交付范围

- 文案与叙事的代码层清理（类名、组件名、注释、品牌资产备注）
- 首页章节编号与 kicker 统一
- 本地偏好接入与占位边界明确化
- 设置入口的路径与动效回归校验
- 残留的 user 概念清理

## 结果要求

- 用户可见文案、代码注释、CSS 类名三层语义一致，至少不再矛盾
- 首页章节编号不再撞号，kicker 风格在同一页面保持同一套口吻
- 偏好开关要么真正影响界面，要么在 UI 上明确标为"占位"
- 全局再搜一次"纺织 / 织 / weave / loom / spindle"，只应剩下品牌资产层的历史命名，并且都有注释说明
- 设置入口齿轮动效在桌面、移动端、`prefers-reduced-motion` 三种情况下都表现合理
