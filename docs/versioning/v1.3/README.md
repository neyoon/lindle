# 版本 1.3

## 定位

1.3 版本聚焦 Flow 的可用性收口。

重点不是继续增加能力，而是降低不会写代码的用户在编辑、理解、运行和排错时的使用成本。

## 目标

- 让 Flow 的关键概念更稳定，不因重命名等普通操作而破坏引用
- 让错误更早暴露，在保存和运行前就给出明确反馈
- 让默认输出和默认数据传递更符合直觉
- 让 Plugin 配置更少依赖手写 JSON 模板
- 让 Flow 的块职责更清楚，但不增加 AI 生成和用户搭建的复杂度

## 交付范围

本版本预计处理以下实现类工作：

- Flow 引用标识与显示名称分离
- Flow 保存前和运行前的统一校验
- Output 块语义收口
- Plugin 输入映射交互优化
- Flow 编辑器关键交互优化
- Flow 结构复杂度与块职责边界优化

## 结果要求

- 不会写代码的用户可以更稳定地搭建和修改 Flow
- 常见错误在保存前或运行前即可发现
- 用户修改显示名称时，不会悄悄破坏下游引用
- Plugin 的默认使用路径不要求用户理解 JSON 模板
- AI 生成 Flow 时，结构既清楚又不过度碎片化
