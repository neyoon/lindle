# 版本管理

## 目标

versioning 用于记录每个版本的范围、交付和后续事项。

它不是发布口号集合，而是版本工作记录。

## 规则

- 一个版本一个目录
- 版本目录内记录版本目标、范围和工作项
- 影响结构或规范的改动，要先进入对应版本文档
- 完成后更新状态，保证版本记录可回看
- 暂时不做但值得做的事项，记录在 `deferred-work.md`
