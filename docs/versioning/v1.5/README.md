# 版本 1.5

## 定位

1.5 版本聚焦现有 Flow 的打磨。

重点不是引入新的流程层，也不是扩大低代码配置面，而是让当前 Flow 更稳定、更容易理解、更容易运行和修正。

## 背景

当前 Flow 已经具备可视化编排、AI 编辑、工具步骤、运行面板和结构化导出能力。

但用户默认路径仍然容易被执行细节打断：

- 输入字段配置过早暴露
- 健康检查更像报错，而不是清晰的状态提示
- AI 编辑可能破坏已有配置
- 运行失败后定位还不够明确
- 工具输入映射仍需要用户理解较多内部概念

1.5 的目标是把这些复杂性向系统内部收拢。

## 目标

- 让用户可以更轻地创建和运行 Flow
- 让输入字段成为高级选项，而不是默认建模入口
- 让 Flow 健康状态在视觉上明确可见
- 让运行失败能定位到具体步骤
- 让 AI 编辑不丢失已有配置
- 让工具步骤更容易正确接收结构化输入

## 交付范围

本版本预计处理以下实现类工作：

- FlowSpec 往返保真
- 输入体验简化
- Flow 健康检查视觉化
- 执行反馈增强
- 工具字段映射打磨
- 保存和运行前的轻量修复引导

## 结果要求

- 用户不需要先理解字段名、字段类型、必填和默认值，也能运行简单 Flow
- 字段配置仍然存在，但默认收进高级选项
- Flow 是否可运行，应在画布和工具栏上形成明确状态
- 健康检查应告诉用户“哪里需要补”，而不是只显示错误文本
- AI 编辑后不应丢失模型选择、工具映射、精确连线和输出结构
- 运行失败时，用户应能看到对应步骤和可行动的修复入口
