# 版本 1.4

## 定位

1.4 版本聚焦 Flow 的测试闭环。

重点不是继续扩展编辑器语义，而是让用户可以从“一个计划”直接进入“生成 Flow、运行 Flow、观察中间步骤、判断结果是否合理”的完整验证路径。

## 目标

- 让 Flow 不只是能生成和运行，还能被系统化测试
- 让用户可以把“测试目标”和“测试样例”显式保存下来
- 让系统能够基于每一步的中间输出判断当前 Flow 是否合理
- 让 AI 可以辅助完成测试判断，而不是只负责生成 Flow
- 让 Flow 的创建和验证形成闭环，而不是停留在一次性生成

## 交付范围

本版本预计处理以下实现类工作：

- Flow 测试计划与测试样例模型
- 从计划到 Flow 的创建链路收口
- 执行时的完整步骤观测与回放
- 基于中间输出的 AI 测试裁决
- 测试结果沉淀、复跑和对比

## 结果要求

- 用户可以先写测试计划，再生成 Flow
- 用户可以直接看到每一步的输入、输出和最终结果
- 系统可以根据中间步骤输出判断测试是否合理
- 测试结果不只是“成功 / 失败”，还应包含失败原因和定位信息
- Flow 的创建、运行、验证可以连续完成，而不需要用户手工做大量中间判断
