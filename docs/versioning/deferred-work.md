# 暂缓事项

## 目标

这个文档记录值得做，但当前版本暂时不做的事项。

它的作用不是堆积想法，而是把已经确认有价值、但当前不进入施工范围的工作单独收起来。

## 使用规则

- 只有确认值得做的事项才写进来
- 已经进入当前版本范围的事项，不再留在这里
- 事项描述要能直接指导后续实现，不写空泛愿景
- 如果某项已经开始进入版本施工，应从这里移到对应版本文档

## 当前暂缓事项

### Flow 导入

当前已经支持导出 `tweak_flow` 结构化清单，但还不支持把该清单重新导入为系统内工作流。

后续值得补上：

- 定义受控导入格式
- 要求包含 `manifest_type`、`manifest_version` 和 `raw_workflow`
- 后端提供导入接口，并按版本做兼容处理
- 前端提供导入入口和失败提示

### Flow 代码导出的结构表达

当前代码导出是一个可运行项目目录，但目录层级还不是 Flow 结构本身的直接映射。

后续值得补上：

- 让导出目录更直接表达 step / block 层次
- 让并行、重复和手动连线关系更容易从文件结构中读出来
- 明确哪些信息由目录表达，哪些信息由 manifest 表达

### Flow 代码导出的保真度

当前代码导出更接近“可运行工程生成”，还不是对原始 Flow 的完全保真还原。

后续值得补上：

- 明确支持和不支持的 Flow 能力边界
- 对 `plugin`、并行、`repeat`、手动连线做更完整映射
- 无法保真时，显式拒绝导出或给出明确降级提示

### Agent / Flow 导出 schema 固化

当前已经有结构化导出，但 schema 说明还没有单独文档化。

后续值得补上：

- 单独定义 Agent 和 Flow manifest 的字段说明
- 明确哪些字段用于展示，哪些字段用于导入恢复
- 明确不同 `manifest_version` 的兼容策略
