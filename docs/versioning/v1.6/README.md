# 版本 1.6

## 定位

1.6 版本引入 Flow Outline。

它的目标是让用户先表达流程大纲，再由系统生成可执行 Flow。

用户不需要一开始就关心输入字段、块类型、工具映射、输出结构和连接细节。

## 背景

1.5 负责把现有 Flow 打磨稳定。

在此基础上，1.6 开始把用户创建 Flow 的默认入口向更自然的结构推进。

用户真正想先表达的是：

- 这个流程要解决什么问题
- 大概分成哪几步
- 每一步要完成什么
- 最后想得到什么

而不是马上决定：

- 这是 collect、process、tool 还是 result
- 输入字段叫什么
- 是否必填
- 输出 key 是什么
- 上游连接到哪个字段

## 目标

- 让用户先搭流程大纲
- 让系统从大纲生成 FlowSpec
- 让 FlowSpec 再收敛成可执行 Flow
- 让输入输出块成为系统生成的执行结构，而不是用户默认关注点
- 保留高级编辑能力，允许用户进入现有块级画布继续精调

## 交付范围

本版本预计处理以下实现类工作：

- 新增 Flow Outline 概念
- 新增大纲创建入口
- 从 Outline 生成 FlowSpec
- 从 FlowSpec 生成现有 Workflow
- 大纲视图与块级画布之间的关系定义
- 默认创建路径切换为“大纲优先”

## 结果要求

- 用户可以用几句话或几条步骤创建 Flow
- 系统可以自动补齐输入、处理、工具和输出结构
- 用户不需要先理解字段和块类型
- 高级用户仍然可以进入块级画布编辑执行细节
- 大纲改动不应破坏已经配置好的高级执行细节
