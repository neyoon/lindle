# 文档总览

## 目标

docs 目录只承载文档，不承载示例代码，不承载实现细节复制品。

文档的职责是说明项目是什么、如何组织、为什么这样设计，以及开发时需要遵守什么规则。

## 结构

- architecture：项目架构、边界、核心概念和运行路径
- development：代码规范、文档规范和开发细节
- versioning：版本变更记录、当前版本范围和后续工作清单

## 阅读顺序

1. 先读 architecture，理解项目结构和核心对象
2. 再读 development，理解开发和写文档的约束
3. 最后读 versioning，了解当前版本的范围和变更

## 文档原则

- 自顶向下
- 简洁
- 明确
- 不重复
- 不写废话
- 不在文档中放代码
