# Flow 健康检查

## 目标

Flow 健康检查用于让用户快速知道当前 Flow 是否可运行、哪里需要补充、哪里已经失败。

它不是错误列表。

它是画布、工具栏和配置面板共同组成的状态系统。

## 核心判断

健康检查只回答三个问题。

- 当前 Flow 能不能运行
- 哪一步需要处理
- 用户下一步应该点哪里修

## 状态分级

### 可运行

Flow 可以直接运行。

视觉表现：

- 工具栏显示可运行状态
- 运行按钮可用
- 画布不显示问题标记

### 需要补充

Flow 结构基本成立，但缺少一些用户可以补上的信息。

典型情况：

- 输入说明为空
- 处理步骤缺少提示词
- 工具步骤缺少必填字段映射
- 输出说明不清楚

视觉表现：

- 工具栏显示需要补充
- 对应步骤显示温和提示
- 运行按钮可以保持可用，但运行前应提示可能影响结果

### 阻塞错误

Flow 当前不能安全运行。

典型情况：

- 引用了不存在的步骤
- 手动连接来源失效
- 工具 ID 缺失
- 必需配置完全缺失
- 结构存在无法自动修复的问题

视觉表现：

- 工具栏显示阻塞状态
- 运行按钮不可直接运行
- 对应步骤使用明确错误状态
- 点击状态后直接打开问题步骤

### 运行后发现问题

Flow 已经开始运行，但某一步执行失败。

视觉表现：

- 失败步骤在画布上变成错误状态
- 运行面板定位到失败步骤
- 最终结果区域说明运行未完成
- 提供重新运行和打开配置入口

## 状态反应

健康检查应尽量用反应代替解释。

例如：

- 错误步骤变红
- 警告步骤变黄或暖色
- 成功步骤变绿
- 运行中步骤显示脉冲或进行中标记
- 工具栏显示整体状态
- 配置面板显示当前步骤最重要的一条修复建议

颜色语义需要稳定：

- 红色表示阻塞错误或运行失败
- 绿色表示完成或可用
- 暖色表示需要补充或可能有风险
- 中性色表示普通结构信息

## 信息层级

同一个问题不要在多个地方重复长文本。

推荐层级：

- 画布：只显示状态
- 工具栏：显示整体摘要
- 右侧配置面板：显示当前步骤的修复建议
- 详情区域：显示完整原因和技术信息

例如，一个工具步骤缺少字段映射时：

- 画布上的工具步骤显示需要补充
- 工具栏显示还有 1 个步骤需要补充
- 配置面板显示缺少哪个输入来源
- 详情区才显示原始校验信息

## 修复入口

每条健康检查结果都应能带用户去修。

建议动作：

- 打开对应步骤配置
- 聚焦缺失字段
- 展开高级选项
- 自动填入建议值
- 忽略非阻塞提示

如果没有修复入口，这条检查就不应作为默认提示暴露。

## 和运行状态的关系

健康检查发生在运行前。

运行状态发生在运行中和运行后。

两者共享视觉语言，但含义不同：

- 健康检查告诉用户结构是否准备好
- 运行状态告诉用户执行是否成功

不要把运行失败简单归类成健康检查失败。运行失败应保留实际执行轨迹。

## 和 1.5 的关系

1.5 中，Flow 健康检查应先实现轻量版本。

优先级：

- 工具栏整体状态
- 画布步骤状态
- 右侧配置面板修复建议
- 运行按钮阻塞提示

暂不做复杂测试报告和批量诊断。
