# Paper & Ink — 设计语言本体

色板、字体、纹理、版面节奏。所有视觉决策的单一事实来源是 `frontend/src/index.css`，这份文档是它的人类可读版。

## 1. 色板

不是调色板，是一个"材料表"。每种颜色都对应一种真实物质。

### 基底——纸

| 语义   | 变量           | 用途                           |
| ------ | -------------- | ------------------------------ |
| 浅纸   | `--paper`      | 全局底色、入口卡底             |
| 暖纸   | `--paper-warm` | 次级区域、hover 底、下拉菜单头 |
| 深纸   | `--paper-deep` | 压在上层的"厚一点的纸"         |
| 卡纸   | `--card`       | 卡片、浮层主体                 |
| 柔卡纸 | `--card-soft`  | 二级卡片、入口卡、运行结果     |

原则：纸之间的对比靠**厚度**（变量阶梯），不靠颜色。任何"高亮条"都不应该用别的色相来打破纸系。

### 主墨——字与压痕

| 语义   | 变量          | 用途                 |
| ------ | ------------- | -------------------- |
| 正墨   | `--ink`       | 标题、重要按钮、文字 |
| 中墨   | `--ink-mid`   | 正文                 |
| 淡墨   | `--ink-soft`  | 说明、次要文字       |
| 浅印   | `--ink-faint` | placeholder、占位符  |

原则：不会出现纯黑 `#000` 和纯白 `#fff`。所有"黑"都是温黑（`--ink`），所有"白"都是暖白（`--paper`、`--card`）。

### 纤维——线

| 语义   | 变量            | 用途                                 |
| ------ | --------------- | ------------------------------------ |
| 主线   | `--line`        | 边框、卡片边缘                       |
| 软线   | `--line-soft`   | 阴影底层（替代传统 box-shadow 模糊） |
| 强线   | `--line-strong` | 虚线分割、强调边界                   |

### 四种专色——情绪

色彩只有四种，每一种都有明确情绪，不得滥用。

| 色      | 正色       | 柔色            | 深色          | 语义                                             |
| ------- | ---------- | --------------- | ------------- | ------------------------------------------------ |
| 铁锈 🔴 | `--rust`   | `--rust-soft`   | `--rust-ink`  | 主强调、AI、交互激活、运行中、Agent              |
| 苔藓 🟢 | `--moss`   | `--moss-soft`   | —             | 成功、完成、输出                                 |
| 霜金 🟡 | `--gold`   | —               | —             | 输入、静态标签                                   |
| 乌木 🟣 | `--bruise` | `--bruise-soft` | —             | 错误、危险、删除                                 |

这套色相之所以只取四种，是因为一块界面上同时出现**三种以上情绪色**就已经超过了"纸面"的承载。超出必须先做裁剪。

### 色彩使用规则

- **同一视觉层级只允许有一种情绪色。** 运行中的 block 用 rust；处理完成的 block 用 moss；两者不会同时出现在同一 block 上。
- **hover 不换色相，只换深浅。** 所有 hover 都走同色系的 soft / ink 变体，不要引入第二种颜色。
- **新元素不允许引入新色变量。** 新增元素先找语义对应的既有色变量，找不到要先在这张表里补充一行，再考虑代码。

## 2. 字体

四种字体，各自负责一件事。

| 字体              | 角色         | 用途                                                         |
| ----------------- | ------------ | ------------------------------------------------------------ |
| **Noto Serif SC** | 中文标题字   | 中文大标题、section 标题、块名、agent 名、workflow 名        |
| **Fraunces**      | 英文展示斜体 | 英文 slogan、品牌词（Lindle / Loom / Spindle）、装饰性副标题 |
| **IBM Plex Mono** | 元信息 / 章节序号 | kicker、`app-kicker`、`stage 01 / ii. / iii.`、标签、stamp |
| **Inter**         | 无衬线正文   | body、button、form 正文                                      |

### 排印规则

- **kicker** —— 所有 section 都以一段大写 mono 字作为开篇（`app-kicker` 类），字号 0.68rem，letter-spacing 0.18em，前面带一条 16px 的铁锈横线。这条线是"纺线落位"的视觉锚。
- **section 标题** —— 使用 `.app-section-title`，`Noto Serif SC`，字重 500。里面出现的 `<em>` 会自动带"笔触下划线"（`::after` 用 `skew + 8px 高 rust-soft 色块模拟荧光笔压痕）。
- **装饰性英文副标题** —— 小段 Fraunces italic，颜色 `--ink-soft`。它不承载信息，只承载节奏，所以永远放在 kicker 和 Noto 大标题之间。

## 3. 纹理

纹理是"这是纸"的唯一证据。

### 全局底纹

`body` 背景是两层 `radial-gradient` 点状叠加，点色 `rgba(45,36,27,0.06)`，两层尺寸错位（16px / 40px），产生"纤维不均匀"的感觉。**不得用纯色填充 body**，否则整个纸系失效。

### 虚线——丝线

分割线、列的分割、workflow 内部列的右边界，全部用 `border-dashed` + `--line-strong`。它在视觉上是"手织的纺线"，不是"功能分隔"。

### 流动线条（running-thread）

在 topbar、editor-toolbar、editor-runbar 底部贴着的那条 1px 虚线，是 `background-image` 加 `background-position` 的无限线性动画（见 [motion.md](./motion.md) §6）。它的存在让纸本身"呼吸"，但速度极慢（2.4–3s 一圈），不构成干扰。

### 阴影——纸的厚度

不用模糊阴影（`blur`）作为主要深度。卡片的厚度靠 `0 2px 0 var(--line-soft)`：一条在下缘叠一片软线。只有强调时才在下方追加非常轻的散光（`... , 0 18px 44px -26px rgba(45,36,27,0.28)`）。

## 4. 版面节奏

一页内容从上到下反复套用这个节奏：

```
kicker（mono 大写小字）
─────
章节标题（中文 Noto + 英文 Fraunces 斜体副标题）
─────
正文 / 卡片（纸）
─────
下一节 kicker
```

每块内容之间的纵向留白比一般网页**显著更大**（往往 120–160px），给纸"喘口气"的空间。这不是 wasted space，是纸本身。

## 5. 不可违反清单

- 不引入**玻璃拟态**（`backdrop-filter: blur()` 除 topbar 外，不允许用于主要内容卡）。
- 不引入**霓虹光效**（任何发光阴影、glow、neon 边框）。
- 不引入**渐变色块**作为内容背景（允许 `linear-gradient` 仅出现在 `running-thread` 和章节标题的笔触压痕 `::after`）。
- 不引入**纯黑 / 纯白**。
- 不引入**圆形主按钮**（除 `user-seal` / `brand-seal` 这类"印章"语义）。按钮圆角一律 `rounded-sm`（约 4px）。
- 不允许同一视觉单元出现**两种以上情绪色**。
