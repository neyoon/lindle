# 设计哲学

## Lindle 是什么

Lindle 是一台 **AI 能力编织工作台**。
名字取自 Loom（织机）与 Spindle（纺锤）：把分散的任务、提示、工具、流程，像纺织那样收敛成结构化、可复用的能力。

视觉语言必须服务这个核心动词：**编织与收敛**。

## 为什么选"纸墨"

初版界面走的是深色 + 青蓝荧光的科技感。它表达"强大"，但不表达"编织"。荧光色在视觉上强调的是"节点发光、数据流动"，而 Lindle 的主张恰恰相反：**节点应该沉下去，让结构浮上来**。

纸与墨提供了另一种修辞：

- **纸**：底层的留白、纤维感、横纹竖纹——承载一切"被编织"的东西。
- **墨**：收敛后的结果，稳定、可读、不闪烁。
- **丝线**：节点之间的关系，细、脆、手工。
- **印章**：某件事被"确认"——模板化、发布、已执行。
- **活字**：内容是被"排印"进去的，不是被贴上去的。

这五种隐喻加起来，正好对应 Lindle 的五件事：

| 视觉修辞 | 对应产品概念                       |
| -------- | ---------------------------------- |
| 纸       | 画布、工作区、背景基座             |
| 墨       | 文字、标题、被确认的状态           |
| 丝线     | Block 之间的连接、章节之间的分割线 |
| 印章     | 品牌标记、用户标记、运行状态徽章   |
| 活字     | 关键标题的逐字落位、章节序号       |

## 三条戒律

**1. 不强调"力量"，强调"结构"。**
不用强色块占据视觉中心。视觉中心永远是"这段东西在编织里处在哪一节"。

**2. 不追求"密度"，追求"节奏"。**
页面可以很空。留白本身在说"这里是纸"。节奏由 kicker、标题、正文、线条、印章五件事轮转形成，不靠堆信息。

**3. 不制造"惊喜"，提供"可预期"。**
动效的作用是让人确认"这一段我读懂了、这一步完成了"，而不是"让界面变得有趣"。

## 什么情况下这套语言会失效

- 当一个页面想强行"讲故事"。Lindle 不讲故事，只提供结构。
- 当一个组件想 CRUD 之外还要花哨地表达"自己很智能"。智能是在行为里证明的，不是在样式里。
- 当动效开始抢走用户对内容的注意力。那说明动效做得太大或太花，回到 [motion.md](./motion.md) 里的六个基础动词重写。

## 和项目其他原则的对应

项目原则（见根 README 与 `docs/architecture`）里反复出现的几条：
"结构优于装饰 / 秩序优于复杂 / 克制优于炫耀"。
Paper & Ink 不是对这几条的"解读"，是它的可视化定义：
每一次加一个渐变色、加一个模态浮层、加一段闪烁光效——都会同时违反这几条原则之一。
视觉语言和产品原则是**同一件事**。
