# 视觉语言

## 目标

这一组文档描述 Lindle 前端采用的 **Paper & Ink（纸墨）** 视觉语言：
它要回答什么、它在界面里长成什么样、它落到代码的哪里。

它不是皮肤。它是产品主张在视觉上的结构。

## 总则

- **编织与收敛**：界面不展示力量，展示结构。每一个元素存在是因为它承担一段结构上的角色，不是装饰。
- **结构优于装饰**：线、留白、印章、活字，都是结构；渐变、玻璃、彩带、光斑，不是。
- **秩序优于繁复**：色板、字体、节奏三者稳定不变，变化只发生在需要强调的瞬间。
- **克制优于炫技**：动效为"被理解"服务，不为"被看见"服务。

## 维护原则

- 色板与字体不要随意扩充；需要新色时，先检查是否能用现有的 `--rust / --moss / --gold / --bruise` 组合表达。
- 新增组件先挑出它在视觉语言里承担的"角色"：卡片（纸）、线（丝）、标记（印章）、文字块（活字）。角色不清晰就先别写样式。
- 动效优先复用 [motion.md](./motion.md) 列出的六种基础动效，避免一对一定制。
